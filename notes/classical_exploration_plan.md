# Classical Exploration Plan for Khantraction

**Date:** 2026-03-28  
**Purpose:** Define the full classical research roadmap for Khantraction after the mapping-centered restart.

---

## 1. Governing viewpoint

Khantraction should not presently be treated as a point-particle program.

The right classical target is:

> a compact structured spacetime-fold object that may behave particle-like from the outside while remaining internally extended, folded, and organized.

This distinction should govern the whole classical exploration.

---

## 2. Central classical question

The main classical question is not whether Khantraction produces a literal point particle. It is:

> what classical properties would make a compact folded spacetime object stable, identifiable, and externally particle-like enough to justify the comparison?

---

## 3. Phase A — Rebuild the parameter foundation correctly

### Goals
1. Treat \(\omega\) as scale.
2. Treat \(\theta,\phi,\rho\) as distinct internal angles.
3. Compare the angular channels symmetrically.
4. Use the clean parameter map and Jacobian as the active foundation.

### Key question
> Are the angular parameters genuinely different classical characteristics?

### Expected outputs
- clean mapping derivations,
- Jacobian comparison work,
- angle-role comparison notes.

---

## 4. Phase B — Rebuild the structured-object picture

### Goals
1. Reconfirm the strongest full-quaternion branch family.
2. Re-express the rich branch as a structured object with:
   - compact external profile,
   - folded/organized interior,
   - core and bulk structure.
3. Re-evaluate concentration and size observables.

### Key observables
- mass half-radius,
- mass 90% radius,
- curvature half-radius,
- curvature 90% radius,
- effective external settling radius,
- core radius,
- soft-region width.

### Key question
> Does Khantraction produce coherent compact structured objects with stable classical identity?

---

## 5. Phase C — Distinct classical characteristics of \(\theta,\phi,\rho\)

### Goals
1. Vary \(\theta\), \(\phi\), and \(\rho\) separately and systematically.
2. Compare how each one changes:
   - shape,
   - concentration,
   - compactness,
   - external profile,
   - internal core/bulk balance.
3. Avoid prematurely collapsing all angular behavior into one channel.

### Key question
> Do the angular variables encode genuinely different object traits?

This is one of the most important classical questions in the new phase.

---

## 6. Phase D — Classical identity and persistence

### Goals
1. Test neighborhood stability around multiple angular sectors.
2. Identify which deformations preserve object identity.
3. Distinguish scale variation from angular and structural variation.
4. Build robust family fingerprints.

### Key question
> What makes one Khantraction object the “same kind” of object in classical terms?

---

## 7. Phase E — External particle-likeness of a structured object

### Goals
1. Determine how the object looks from the outside.
2. Compare whether internally different objects become externally similar.
3. Define external particle-likeness without requiring pointlikeness.

### Key observables
- asymptotic profile similarity,
- compactness-like ratios,
- concentration radii,
- settling scales,
- external indistinguishability classes.

### Key question
> Can a folded extended object still behave particle-like externally?

---

## 8. Phase F — Classical hosting properties

### Goals
1. Study whether the structured object can host externally induced content.
2. Compare opposite signed induced loadings.
3. Locate where hosting sensitivity lives.
4. Test whether different angular sectors host content differently.

### Key question
> Can the structured object classically act as a host for externally supplied content?

This is the correct classical precursor to later charge-hosting work.

---

## 9. Phase G — Classical rotational / handedness properties

### Goals
1. Study whether angular sectors define classical chirality classes.
2. Compare whether opposite angular sectors are equivalent, mirrored, or distinct.
3. Identify whether handedness belongs to the object’s classical identity.

### Key question
> Does the structured object have a classical handedness architecture?

This is the correct classical precursor to later spin-like work.

---

## 10. Phase H — Only then return to stronger quantum-facing work

Once the classical object and the distinct angular roles are rebuilt carefully, then quantum-facing work can restart on stronger footing.

That later stage can ask:
- whether excitation channels exist,
- whether mode ladders form,
- whether chirality or loading affect them,
- and whether quasi-discrete structure survives improved operators.

But that should come **after** the classical object picture is rebuilt symmetrically across the angular variables.

---

## 11. Condensed roadmap

The whole classical program can be summarized in five main questions:

1. **Mapping** — What are \(\omega,\theta,\phi,\rho\) really doing?
2. **Objecthood** — Does Khantraction produce coherent compact structured objects?
3. **Distinct angular traits** — Do \(\theta,\phi,\rho\) generate genuinely different characteristics?
4. **Identity and persistence** — What makes these objects classically stable and classifiable?
5. **External particle-likeness** — Can they behave particle-like externally while remaining internally extended?

---

## 12. Immediate next task

The first active task of the new classical phase should be:

> compare \(\theta\), \(\phi\), and \(\rho\) symmetrically as classical characteristics.

That is the most important unresolved foundation issue after the mapping-centered restart.

---

## 13. Bottom line

**Bottom line:** the classical exploration of Khantraction should now proceed as a structured-object program, not a point-particle program. Its next phase must rebuild the theory from the clean parameter mapping and treat \(\theta,\phi,\rho\) symmetrically before any deeper interpretation becomes canonical.
